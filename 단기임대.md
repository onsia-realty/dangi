# BOOIN 단기임대 플랫폼 MVP 구축 지시서

> 이 문서 전체를 Claude Code CLI에 프롬프트로 입력한다.
> 작성일: 2026-07-07 / 발주: 온시아(주)

---

## 1. 프로젝트 배경 (컨텍스트)

- 벤치마크: 삼삼엠투(33m2.co.kr) — C2C 직거래 단기임대 플랫폼 (1주 단위 계약, 고정 보증금, 에스크로)
- **우리의 차별점: "공인중개사 검증 단기임대"**
  - 33m2는 임대인 직접 등록 → 전대사기 리스크, 계약서·세금계산서·법인 증빙 불가
  - 우리는 공인중개사(온시아)가 매물을 검증하고 계약을 책임지는 구조
- 공급 확보 방식: 공실클럽 등 기존 채널에서 공실 매물 발굴 → 임대인/등록 중개사에게 단기임대 전환 제안 → **동의받은 매물만** 등록 (제안→동의→등록 파이프라인)
- 운영 주체: 온시아공인중개사사무소 (초기에는 온시아 단독 운영, 추후 파트너 중개사 확장)

## 2. 개발 원칙 (반드시 준수)

Karpathy 4원칙으로 작업한다:
1. **Think Before Coding** — 코드 작성 전 구조 먼저 제시하고 확인받기
2. **Simplicity First** — MVP 범위 밖 기능은 만들지 않는다. 과설계 금지
3. **Surgical Changes** — 수정 시 관련 파일만 최소 변경
4. **Goal-Driven Execution** — 각 단계 완료 기준을 명확히 하고 진행

추가 규칙:
- 실제로 **돌아가는 껍데기(working skeleton)**가 목표. 화면 이동, 등록, 조회, 문의가 실제로 작동해야 함
- 더미 데이터 시드 스크립트 포함 (매물 10건 정도)
- 모든 UI 텍스트는 한국어
- 모바일 우선 반응형 (이용자 대부분 모바일)

## 3. 기술 스택 (고정)

- **Next.js 15 (App Router) + TypeScript**
- **Supabase** — DB, Auth(관리자 로그인), Storage(매물 사진)
- **Tailwind CSS**
- 배포 타깃: **Vercel**
- 지도: 네이버 지도 SDK (기존 booin.co.kr 인증 경험 있음) 또는 카카오맵 중 구현 쉬운 것. MVP는 매물 상세에 지도 1개만
- 결제/전자계약/에스크로는 **MVP에서 구현하지 않음** (Phase 2)

## 4. MVP 범위 — Phase 1

### A. 공개 사이트 (임차인용)

| 페이지 | 경로 | 내용 |
|---|---|---|
| 홈 | `/` | 서비스 소개 히어로("공인중개사가 검증한 단기임대"), 추천 매물 카드, 이용 흐름 3단계, 문의 CTA |
| 매물 목록 | `/rooms` | 카드 리스트 + 필터(지역, 주간 요금대, 건물유형, 입주가능일) |
| 매물 상세 | `/rooms/[id]` | 사진 슬라이드, 주간 요금·보증금·관리비, 옵션 아이콘, 입주 가능 캘린더, 지도, **검증 뱃지**, **시세 비교 배지**, 계약 문의 폼 |
| 이용 안내 | `/guide` | 계약 절차, 보증금·정산 안내, FAQ |
| 임대인/중개사 안내 | `/partner` | "공실을 단기임대로 수익화" 제안 페이지 + 매물 접수 폼 |

핵심 UI 요소:
- **검증 뱃지**: "공인중개사 검증 완료" (등기부 확인 여부 admin에서 체크 시 노출)
- **시세 비교 배지**: 관리자가 입력한 주변 월세 시세 대비 적정가 표시. 예: "주변 월세 환산가 대비 -8%" (MVP는 수동 입력값 기반, 국토부 실거래 API 자동화는 Phase 2)
- 주간 요금 체계: 1주 단위 계약, 요금 = 주간요금 × 주수, 보증금 고정(기본 33만원, 매물별 변경 가능)

### B. 관리자 (온시아 내부용) — `/admin` (Supabase Auth 로그인 필수)

1. **매물 관리**: 등록/수정/숨김. 필드는 아래 DB 스키마 참조. 사진 다중 업로드(Storage)
2. **리드 파이프라인** (핵심 기능): 공실 매물 발굴→전환 트래킹 칸반 또는 테이블
   - 상태: `발굴` → `제안발송` → `협의중` → `동의완료` → `등록완료` / `거절`
   - 필드: 매물 주소, 소유자/담당 중개사 연락처, 출처 메모(내부용, 공개 안 됨), 제안일, 동의일, 동의 방식 메모, 담당자
   - **동의완료 상태의 리드만 매물로 전환(버튼 1개로 properties 생성)**
3. **문의 관리**: 계약 문의 리스트, 상태(신규/연락완료/계약진행/완료/취소), 메모

### C. 계약 흐름 (MVP)

온라인 결제 없음. 흐름:
1. 임차인이 매물 상세에서 **계약 문의 폼** 제출 (이름, 연락처, 희망 입주일, 기간)
2. admin 문의함에 알림 → 온시아가 전화/카톡으로 상담
3. 계약은 오프라인/기존 전자계약 도구로 진행, admin에서 상태만 관리
4. 계약 확정 시 해당 매물 캘린더에 예약 기간 블록 처리

## 5. DB 스키마 (Supabase)

```sql
-- 매물
properties (
  id uuid pk, created_at,
  title text, address text, address_detail text,  -- address_detail은 계약 확정 전 비공개
  lat float, lng float,
  building_type text,        -- 원룸/오피스텔/아파트/기타
  weekly_rent int,           -- 주간 요금(원)
  deposit int default 330000,
  mgmt_fee int,              -- 주간 관리비 or 0
  min_weeks int default 1,
  options jsonb,             -- {wifi, washer, aircon, ...}
  photos text[],             -- storage URLs
  verified boolean default false,      -- 등기부 확인 완료
  market_monthly_rent int,             -- 주변 월세 시세(수동입력, 시세배지 계산용)
  status text default 'active',       -- active/hidden/booked
  lead_id uuid fk nullable
)

-- 리드 (공실 발굴 파이프라인)
leads (
  id uuid pk, created_at,
  address text, owner_name text, owner_contact text,
  source_memo text,          -- 출처/경위 메모(내부 전용)
  stage text default '발굴', -- 발굴/제안발송/협의중/동의완료/등록완료/거절
  proposed_at date, agreed_at date, agree_memo text,
  assignee text
)

-- 문의
inquiries (
  id uuid pk, created_at,
  property_id uuid fk,
  name text, phone text,
  move_in date, weeks int,
  message text,
  status text default '신규' -- 신규/연락완료/계약진행/완료/취소
)

-- 예약 블록 (캘린더)
bookings (
  id uuid pk, property_id uuid fk,
  start_date date, end_date date,
  inquiry_id uuid fk nullable, memo text
)
```

RLS: 공개 테이블은 properties(status='active')와 inquiries INSERT만 anon 허용. leads·bookings·admin 기능은 인증 필수.

## 6. 시세 비교 배지 로직 (MVP)

```
월세 환산가 = market_monthly_rent / 4.345   -- 주간 환산
차이율 = (weekly_rent - 월세환산주간가) / 월세환산주간가 × 100
표시: 차이율 ≤ +15% → "합리적 요금" 초록 배지 + "주변 월세 환산가 대비 {±n}%"
      그 외 → 배지 미표시 (부정 표시 안 함)
market_monthly_rent 미입력 시 배지 자체 미노출
```
